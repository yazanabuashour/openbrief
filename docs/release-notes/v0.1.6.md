# OpenBrief v0.1.6

This patch release adds previous-brief context to runner output, hardens maintainer validation, and records OpenBrief taste-review decisions while preserving the existing runner commands and install flow.

## Changed

- Added `previous_briefs` to brief runner JSON so agents can see recent delivered brief context without inspecting private storage directly.
- Split runner, fetch, delivery, health, source normalization, and SQLite internals into smaller modules with focused tests.
- Broadened release and pull-request skill validation to run all agent skill validators through `scripts/validate-all-agent-skills.sh`.
- Added maintainer taste-review guidance, candidate-surface comparison records, and AgentOps surface policy docs for future ADR, POC, eval, and promotion decisions.

## Verification

- Local release gates passed: release-doc validation, skill validation, formatting check, golangci-lint, and unit tests.
