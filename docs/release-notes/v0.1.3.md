# OpenBrief v0.1.3

This patch release refines feed processing for configured RSS and Atom sources without changing the runner CLI, JSON schema, SQLite schema, or install flow.

## Changed

- Bound network-backed feed URL canonicalization per source so slow or unresolved redirects cannot dominate an entire brief run.
- Preserve feed item ordering while resolving supported URL canonicalization strategies in parallel.
- Keep all items from truncated canonicalized feeds eligible for new-item selection so skipped canonicalization attempts do not hide later feed entries.
- Make RSS `<source>` outlet extraction effective for candidate outlet fields and outlet policy matching when sources use `outlet_extraction: "rss_source"`.

## Verification

- Local release gates passed: release-doc validation, skill validation, formatting check, golangci-lint, and unit tests.
