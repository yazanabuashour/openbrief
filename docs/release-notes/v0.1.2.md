# OpenBrief v0.1.2

This patch release refines the public OpenBrief skill contract and release validation coverage without changing the runner CLI, JSON schema, SQLite schema, or install flow.

## Changed

- Allow user-directed legacy automation/config migration through a reviewed draft-and-apply OpenBrief skill workflow.
- Harden skill markdown validation tests so release checks focus on the supported policy contract instead of brittle prose details.

## Verification

- Local release gates passed: release-doc validation, skill validation, formatting check, golangci-lint, and unit tests.
- Targeted production agent eval scenarios passed: routine-agent-hygiene, invalid-source-config-rejects, and rss-source-first-run-candidate.
