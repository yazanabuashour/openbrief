# OpenBrief v0.1.8

This patch release moves brief-history handoff onto the `record_delivery` result so agents can send the just-recorded brief plus the two latest prior deliveries without reconstructing history from large `run_brief` payloads.

## Changed

- Added `deliveries` and `final_answer` to `record_delivery` results, populated from the runner-owned delivery table after the current brief is recorded.
- Updated the OpenBrief skill contract to answer with `record_delivery.final_answer` when present and keep `run_brief.previous_briefs` as a legacy fallback.
- Kept delivery recording durable and retry-safe by returning a current-brief final answer even if the post-write history read fails.
- Added runner and skill regression tests for latest-three delivery history, exact recorded-message preservation, `NO_REPLY`, health footnotes, and post-write history read failures.

## Verification

- Local release gates passed: release-doc validation, skill validation, formatting check, golangci-lint, unit tests, and the targeted brief-run-history production-agent eval.
