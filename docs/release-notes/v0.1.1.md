# OpenBrief v0.1.1

This patch release refines the public OpenBrief skill contract without changing the runner CLI, JSON schema, SQLite schema, or install flow.

## Changed

- Renamed the shipped agent skill references to the `OpenBrief` display name across docs and validation tests.
- Narrowed the production runner-bypass refusal policy so routine OpenBrief tasks still reject unsupported direct data access while repository development, docs review, tests, release verification, security review, and migration design can inspect public repository files.

## Verification

- Local release gates passed: release-doc validation, skill validation, formatting check, golangci-lint, unit tests, and committed-doc policy scrub.
