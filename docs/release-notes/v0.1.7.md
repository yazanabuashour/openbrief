# OpenBrief v0.1.7

This patch release fixes previous-brief rendering guidance so agents preserve recorded brief bodies and links while keeping the runner CLI, JSON schema, SQLite schema, and install flow unchanged.

## Changed

- Updated the OpenBrief skill contract to require each `previous_briefs[].message` body to be rendered exactly as recorded, preserving Markdown links, `NO_REPLY`, and health footnote text.
- Strengthened the brief-history production eval verifier and skill regression tests so lossy previous-brief summaries no longer pass.
- Refreshed maintainer instructions from the OpenClerk workflow template with OpenBrief-specific safety language.

## Verification

- Local release gates passed: release-doc validation, skill validation, formatting check, golangci-lint, unit tests, and the targeted previous-brief rendering test set.
